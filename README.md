# unifying_simulation — AOG 자재 수급 어시스턴트

AOG(Aircraft on Ground) 상황 발생 시, 정해진 절차에 따라 자재를 수급하고 각 단계에서
실제로 연락/발송/조회까지 이어갈 수 있도록 돕는 Streamlit 대시보드입니다.
Google Colab의 노트북 하나(`AOG_Dashboard.ipynb`)로 배포됩니다.

## 실행 방법

1. `AOG_Dashboard.ipynb`를 [Google Colab](https://colab.research.google.com/)에서 엽니다.
2. **Cell 1 → Cell 2 → Cell 3 순서대로** 실행합니다.
   - Cell 1은 설치 직후 런타임을 자동 재시작합니다. *"세션이 다운되었습니다"* 경고가 떠도 정상이니 무시하고 Cell 2부터 이어서 실행하세요.
   - Cell 3은 `google.colab.output`으로 앱을 새 창에 띄웁니다. npm/localtunnel 같은 제3자 터널을 전혀 쓰지 않으므로 사내망 보안 정책에 걸릴 일이 없습니다.
3. 새 창에서 대시보드가 열리면 사이드바에서 **"🚨 AOG 대시보드"** 또는 **"⚙️ 데이터 관리"** 화면을 선택합니다.

## 화면 구성

### 🚨 AOG 대시보드 (메인 화면)
- 상단: **항공기 기종 / 자재 넘버 / AOG 발생 공항**을 드롭다운이 아닌 **직접 입력(텍스트)** 으로 받습니다. 대소문자·공백 차이는 자동으로 정규화되어 매칭됩니다.
- 좌측 절반 — **🤖 AI 상황 요약 & 추천 행동**: 지금까지 조회된 단계별 결과와 다음에 취해야 할 행동을 규칙 기반으로 즉시 요약합니다(항상 동작). 원하면 "🧠 로컬 LLM으로 브리핑 다듬기"를 펼쳐 로컬 모델(Qwen2.5-0.5B-Instruct)로 문장을 다듬을 수 있습니다 — 모델 로딩에 실패해도 규칙 기반 요약은 그대로 유지되어 대시보드가 절대 멈추지 않습니다.
- 우측 절반 — **📋 프로세스 진행**: 현재 단계 결과를 보여주고, **"▶ 다음 단계 진행"** 또는 **"✅ 현재 단계에서 조치(프로세스 종료)"** 를 사용자가 직접 눌러야 다음으로 넘어갑니다(자동 진행 없음). 6단계(Hand-carry)는 최후 수단이라 "다음 단계" 버튼 자체가 없습니다.

### 단계별 종료 시 실제로 연결되는 조치
| 종료 단계 | 제공되는 조치 |
|---|---|
| 1단계 FAK | 케이스 시작 시 입력한 **요청 정비사 연락처**로 바로 걸 수 있는 `tel:` 링크 제공 |
| 2단계 Allocation | 해당 공항의 **Allocation 부서** 전화(`tel:`)·메일(`mailto:`) 링크 제공 |
| 3단계 Pooling | 매칭된 **Pooling 파트너사** 전화·메일 링크 제공 |
| 4·5단계 (Main Station 타사 / 동일기종 타사) | 조회 대상 항공사 **전원**의 이메일을 수신자로 채운 `mailto:` 긴급 메일이 자동 작성됨(제목/본문 템플릿 포함), 발송 후 "발송 완료로 기록" 가능 |
| 6단계 Hand-carry | **통관팀** 연락처 + AOG 공항행 **가장 빠른 편명 후보**(실시간 시간 계산)를 보여주고, 선택한 편으로 확정 기록 가능 |

각 조치는 처리 이력(Audit Log)에 자동으로 남습니다.

### ⚙️ 데이터 관리 (별도 화면)
FAK/Allocation 재고, Pooling 계약, 공항별 Main Station 항공사, 기종별 운영 항공사,
Allocation 부서·통관팀 연락처, Hand-carry 편명 스케줄까지 — 표 형태로 직접 추가/수정/삭제하고
**"💾 저장"** 을 누르면 `aog_db.json`에 즉시 반영됩니다. 노트북을 껐다 켜도 마지막 저장 내용이 유지됩니다.
(주기적으로 바뀌는 FAK 구성품, 공항별 Pooling 계약사 등을 비개발자도 손쉽게 갱신할 수 있도록 만든 화면입니다.)

## 테스트 시나리오

값을 그대로 입력해 각 단계에서 프로세스가 어떻게 종료되고 어떤 조치가 뜨는지 확인할 수 있습니다.
(항공기 기종 / 자재 넘버 / AOG 발생 공항 순, 대소문자·공백은 무관합니다)

### 시나리오 1 — 1단계 FAK에서 즉시 해결 → 정비사 재연락
- `A330-300` / `OXY-GEN-A330-15` / `ICN`
- "요청 정비사 연락처"에 아무 번호(예: `010-1234-5678`)를 입력해두면, 종료 후 그 번호로 바로 거는 `tel:` 링크가 뜹니다.

### 시나리오 2 — 2단계 Allocation까지 가야 해결 → Allocation 부서 연락
- `B777-300ER` / `BRK-B777-CARBON-01` / `ICN`
- FAK 없음 → 다음 단계 진행 → Allocation 확인 → 종료 시 ICN 지점 Allocation 부서 연락처가 뜹니다.

### 시나리오 3 — 3단계 Pooling 파트너사까지 가야 해결 → 파트너사 연락
- `B737-800` / `HYD-PUMP-737-11` / `SIN`
- FAK·Allocation 없음 → Pooling에서 **SIA Engineering** 확인 → 종료 시 해당 파트너사 연락처가 뜹니다.

### 시나리오 4 — 끝까지 가서 Hand-carry로 종료 → 통관팀 + 편명 조회
- `A321neo` / `FMS-A321-NEO-09` / `CDG`
- 1~5단계 모두 없음("다음 단계 진행"을 4번) → 6단계에서 통관팀 연락처와 CDG행 가장 빠른 편명 후보가 뜹니다. 원하는 편을 선택하고 확정할 수 있습니다.

### (참고) 보너스 시나리오 — 4·5단계 해결, 긴급 메일 작성
- **4단계 해결**: `A321neo` / `CAB-DOOR-ACT-321-03` / `GMP` → 1~3단계 없음 → 4단계에서 GMP Main Station 항공사(제주항공) 확인 → 종료 시 GMP를 Main Station으로 쓰는 항공사 **전원**에게 보내는 긴급 메일이 자동 작성됩니다.
- **5단계 해결**: `B737-800` / `WHL-NLG-737-22` / `ICN` → 1~4단계 없음 → 5단계에서 B737-800 운영사(티웨이항공) 확인 → 종료 시 B737-800 운영 항공사 **전원**에게 보내는 긴급 메일이 자동 작성됩니다.

### 입력 오류 / 수동 조기 종료 테스트
- 값을 하나라도 비운 채 시작하면 경고만 표시되고 프로세스는 시작되지 않습니다.
- 재고가 없는 단계에서도 "현재 단계에서 조치"를 누르면 별도 조치 버튼 없이 "수동 종료" 안내만 표시되며 정상적으로 이력에 남습니다.

### 데이터 관리 화면 테스트
- "⚙️ 데이터 관리"로 이동해 아무 표(예: FAK 재고)에 행을 추가하고 "💾 저장"을 누른 뒤, "🚨 AOG 대시보드"로 돌아와 방금 추가한 기종/자재 넘버로 케이스를 시작하면 1단계에서 바로 확인되는 것으로 변경 사항이 즉시 반영됨을 확인할 수 있습니다.
