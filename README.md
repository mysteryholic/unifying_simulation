# unifying_simulation — AOG 자재 수급 대시보드

AOG(Aircraft on Ground) 상황 발생 시, 정해진 절차에 따라 자재를 수급할 수 있도록 돕는
Gradio 기반 단일 셀 Colab 대시보드입니다.

## 실행 방법

1. `AOG_Dashboard.ipynb`를 [Google Colab](https://colab.research.google.com/)에서 엽니다.
   (Colab 상단 메뉴 `파일 → 노트북 열기 → GitHub` 탭에서 이 저장소를 검색해도 됩니다.)
2. 코드 셀 **하나**를 실행(Shift+Enter)합니다. `!pip install -q gradio` 설치 후 대시보드가
   셀 출력 안에 자동으로 인라인 표시됩니다. `share=True`나 localtunnel 같은 제3자 터널을
   전혀 쓰지 않으므로 사내망 보안 정책에 걸릴 일이 없습니다.
3. 화면에서 **항공기 기종 / 자재 넘버(Part Number) / AOG 발생 공항**을 선택하고
   **"🚨 AOG 프로세스 시작"** 버튼을 누릅니다.

## 업무 프로세스

프로그램은 자재를 찾을 때까지 아래 순서를 단계별로 진행하며, 각 단계 결과를 보여준 뒤
사용자가 버튼으로 직접 다음 동작을 선택해야 진행됩니다(자동 진행 없음).

| 단계 | 내용 |
|---|---|
| 1 | 당사 FAK(Fly Away Kit) 자재 재고 확인 |
| 2 | 당사 Allocation 자재 재고 확인 |
| 3 | Pooling 계약 파트너사 재고 확인 |
| 4 | 해당 공항을 Main Station으로 쓰는 타 항공사 대여 요청 |
| 5 | 동일 기종을 운영하는 타 항공사 대여 요청 |
| 6 | 본사 직원 Hand-carry 이동 (최종 수단, 항상 조치 가능) |

각 단계 결과 화면에는 다음 두 버튼이 함께 표시됩니다.

- **▶ 다음 단계 진행** — 현재 단계에서 재고를 찾지 못했을 때, 다음 단계 조회로 넘어갑니다. (6단계에서는 더 이상 다음 단계가 없으므로 숨겨집니다.)
- **✅ 현재 단계에서 조치 (프로세스 종료)** — 재고를 찾았거나, 담당자가 현재 단계 결과로 상황을 종결하기로 했을 때 누르면 프로세스가 즉시 종료됩니다.

## 더미 데이터 구성

코드 상단에 아래 더미 데이터가 정의되어 있습니다 (실제 운영 시에는 사내 재고관리시스템 API 조회로 대체).

- `FAK_STOCK`, `ALLOCATION_STOCK` — 기종/자재넘버/수량/위치
- `POOLING_PARTNERS` — 파트너사별 연락처 및 보유 재고
- `MAIN_STATION_AIRLINES`, `OTHER_AIRLINE_STATION_STOCK` — 공항별 Main Station 운영 항공사 및 대여 가능 재고
- `FLEET_OPERATORS`, `OTHER_AIRLINE_FLEET_STOCK` — 기종별 운영 항공사 및 대여 가능 재고

## 테스트 시나리오

항공 자재 지식이 없어도 아래 값을 그대로 입력해 각 단계에서 프로세스가 어떻게 종료되는지
바로 확인할 수 있습니다. (항공기 기종 / 자재 넘버 / AOG 발생 공항 순)

### 시나리오 1 — 1단계 FAK에서 즉시 해결
- 항공기 기종: `A330-300`
- 자재 넘버: `OXY-GEN-A330-15`
- AOG 발생 공항: `ICN`
- **예상 결과**: 1단계(FAK)에서 바로 재고(2개, ICN FAK 창고 A-12)가 확인되어, "다음 단계 진행" 버튼을 누를 필요 없이 "현재 단계에서 조치"로 즉시 종료할 수 있습니다.

### 시나리오 2 — 2단계 Allocation까지 가야 해결
- 항공기 기종: `B777-300ER`
- 자재 넘버: `BRK-B777-CARBON-01`
- AOG 발생 공항: `ICN`
- **예상 결과**: 1단계(FAK)에서는 재고 없음 → "다음 단계 진행" 클릭 → 2단계(Allocation)에서 재고(1개) 확인 → 종료.

### 시나리오 3 — 3단계 Pooling 파트너사까지 가야 해결
- 항공기 기종: `B737-800`
- 자재 넘버: `HYD-PUMP-737-11`
- AOG 발생 공항: `SIN`
- **예상 결과**: 1단계 FAK 없음 → 2단계 Allocation 없음 → 3단계에서 Pooling 파트너사 **SIA Engineering(싱가포르)**의 재고(1개, SIN 창고)를 확인 → 종료.

### 시나리오 4 — 끝까지 가서 Hand-carry로 종료
- 항공기 기종: `A321neo`
- 자재 넘버: `FMS-A321-NEO-09`
- AOG 발생 공항: `CDG`
- **예상 결과**: 1~5단계 모두 재고 없음("다음 단계 진행"을 4번 클릭) → 6단계(Hand-carry)에서 본사 직원이 자재를 직접 들고 이동하는 것으로 최종 조치. 6단계는 최후의 수단이라 "다음 단계 진행" 버튼이 아예 표시되지 않고 "✅ Hand-carry 조치 확정" 버튼만 남습니다.

### (참고) 보너스 시나리오 — 4·5단계에서 해결되는 케이스
- **4단계(Main Station 타 항공사)에서 해결**: 기종 `A321neo`, 자재 넘버 `CAB-DOOR-ACT-321-03`, 공항 `GMP` → 1~3단계 없음 → 4단계에서 GMP를 Main Station으로 쓰는 **제주항공** 재고 확인 → 종료.
- **5단계(동일 기종 타 항공사)에서 해결**: 기종 `B737-800`, 자재 넘버 `WHL-NLG-737-22`, 공항 `ICN` → 1~4단계 없음 → 5단계에서 B737-800 운영사인 **티웨이항공** 재고 확인 → 종료.

### 입력 오류 테스트
기종/자재넘버/공항 중 하나라도 비워둔 채 "🚨 AOG 프로세스 시작"을 누르면, 프로세스가 시작되지 않고 값을 모두 입력하라는 경고 메시지만 표시됩니다.
